Test Pilot: Tab Open/Close Study: Aggregated Data Samples

Test Description

  • Test name: Tab Open/Close study .
  • Test duration: 1 week*
  • Test type: Global
  • Firefox versions covered: 3.5 only
  • Data submission: over 5000 test sets submitted from 50+ locales, September 3rd - 24th, 2009.

* (i.e. for each user, data collection began when they installed the extension and ended 1 week later)

Test Results

We are working on analyzing the data, and our results will be available online soon. For example, we've used the data to estimate the average number of tabs that are open in Firefox at a time, as well as the length of time that the average tab remains open.

There's a lot more we can do with this data. If you have questions about it, ideas for how to analyze it, or derived metrics that you would be interested in seeing, please join the Test Pilot Discussion Group . If you generate any statistics or charts from the data, we would of course look forward to seeing those posted in the discussion group as well.

How to read the data

The data is in CSV (comma-separated value) format. The first line of the file contains the column names. A chunk of sample data might look like this: 
			user, event_code, tab_position, tab_window, ui_method, tab_site_hash, num_tabs, timestamp
			1, 6, 3, 1, 0, 1, 4, 1252974367669
			1, 5, 0, 1, 1, 0, 4, 1252974422507
			1, 10,0, 1, 0, 0, 0, 1252974427743
			1, 9, 0, 1, 0, 0, 1, 1253668079410
			1, 6, 0, 1, 0, 0, 1, 1253668079792
			1, 1, 1, 1, 2, 0, 2, 1253668084072
			1, 5, 1, 1, 2, 0, 2, 1253668084102
			1, 1, 2, 1, 2, 0, 3, 1253668084427
			1, 5, 2, 1, 2, 0, 3, 1253668084471
			1, 1, 3, 1, 2, 0, 4, 1253668084531
			1, 5, 3, 1, 2, 0, 4, 1253668084568
Each row represents a single event in which a user interacted with the Firefox tabs bar. Here's what the columns mean.
  1. user: All rows that have the same user number came from the same submission. Each submission is assumed to be a different user, but there are various ways that a single user could make multiple submissions (e.g. multiple computers, multiple Firefox profiles, etc.) These user numbers have no meaning or correlation to anything outside of the aggregated data file in which they appear. Their order has been randomized, so the numbers do not correspond to the order in which submissions were received.
  2. event_code: Can be interpreted using the following table:
    1. (1) OPEN_EVENT: a new tab was opened.
    2. (2) CLOSE_EVENT: a tab was closed.
    3. (3) DRAG_EVENT: user began dragging a tab away from its current location.
    4. (4) DROP_EVENT: a dragged tab was dropped into a new location.
    5. (5) SWITCH_EVENT: the tab became the active tab, either because the user switched to it manually, or because of automatic browser behavior (i.e. opening a new tab usually generates an automatic switch to that tab.)
    6. (6) LOAD_EVENT: the URL displayed in the tab changed, whether because of a click on a link, a URL entered into the location bar, etc.
    7. (7) Not used. (Reserved for future use.)
    8. (8) Not used. (Reserved for future use.)
    9. (9) OPEN_WINDOW_EVENT: a new browser window was opened.
    10. (10) CLOSE_WINDOW_EVENT: an open browser window was closed.
  3. tab_position: the position of the affected tab within the window. They are numbered left to right, starting at 0. Be aware that these position numbers are ephemeral: the position of a tab can change several times between when it is opened and when it is closed, due to drag-drop events, or due to tabs being opened or closed to the left of the tab in question. If you want to track a single tab over time, you will need to keep track of changes in its position.
  4. tab_window: a number identifying the window to which the tab belongs. The first window opened during the session will be numbered 1, and the window ids increase from there.
  5. ui_method: Reserved for future use. Ignore the numbers in this column for now.
  6. tab_site_hash: Identifies the 'tab group' to which the tab belongs. Tabs which have the same number in this column are open to pages in the same domain. For example, if a user had three tabs open to different pages on the www.mozilla.org site, those three tabs would all have the same number in the tab_site_hash column, but a tab open to a site beginning with labs.mozilla.com would have a different number. The actual number is an index into a client-side hash of the domain name, so it should not be possible to recover from this number any information about actual URLs visited.
  7. num_tabs: The number of tabs which were open in the same window after this event was processed. It's per-window, so if you want the number of tabs open in all windows you will need to sum this value across multiple windows. If the event changed the number of tabs (e.g. OPEN_EVENT, CLOSE_EVENT) then the number in num_tabs represents the state after that event.
  8. timestamp: The time at which the event occured, given in milliseconds since epoch.
