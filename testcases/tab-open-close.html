Tab Open/Close Study

  • Analysis of data
  • Aggregated data samples for download

Introduction

Tabbed browsing is now the standard interface offered by all major web browsers, and it‘s been argued that the browsing experience becomes difficult to manage when more than a handful of tabs are opened. For this test, we would like to explore what would be the best default browser behavior after users close or open a tab when many tabs are open. We will simply log user action types and time stamp when they interact with tabs. Through these log data, we may be able to detect certain patterns that help us design the default browser behavior that works best for most users.   

One thing to note is that Firefox tab behaviors may be dramatically changed by different tab-related add-ons. In order to distinguish test data from browsers with such add-ons and browsers without them, the data that is uploaded will include information about add-ons that are installed.

Test Champion: Alexander Limi , User Experience Designer in Mozilla
Test Duration: 1 week
Test Type: Global.  

This test is not localized to any other languages yet. If you have any questions for this test or are interested to help us improve test metrics, please join us in the Test Pilot discussion group.

Test Detail

 

Question 1: What would be the best default tab behavior after users close a tab?  

Background: 

In the past, we have observed that people close a tab in different contexts, for example:   

  • When people finish using the tab: e.g. users close the tab when they finish watching a video. 
  • When people organize browser tabs: e.g. users close some less important or less used tabs when there are too many tabs open in a browser. 
  • When people temporarily hide a tab: e. g. users close a tab when he detects someone is passing by; he would like to open it again when nobody is watching. 

    • For different contexts, people have different intentions after closing a tab, for example:

  • open another interesting tab
  • return to a task in progress in another tab
  • close another less important or less used tab
  • open whatever tab
  • open the most often used tab
    •  

    Main Data Type: 

    To understand user interactions with current Fx design, we will record and timestamp interactions that open, close, or switch tabs.  We will not record the URLs that are loaded in any tab; we will record only the fact of whether or not two tabs belong to the same site, in order to see whether this makes a difference.

     

    Question 2:  What should be default placement when users open a new tab?   

    We have observed that people open a tab for different reasons, for example:

  • Open a new blank tab to type URL or to perform search
  • Open a new tab through links within current tab

    • We also notice that people do one of the following actions after they open a new tab:

  • drag the newly opened tab next to a particular tab 
  • drag the newly opened tab to a new window

    • Main Data Type:  

    For this test, we would like to explore what we can do to support people open a new tab in a meaningful way. We will record and timestamp tab open, close, and drag-and-drop events.
