A Week in the Life of a Browser: Aggregated Data Samples

Test Description

  • Test name: A Week in the Life of a Browser.
  • Test duration: 1 week, repeating every 60 days
  • Test type: Global
  • Firefox versions covered: 3.5 and above
  • Data submission: 6860 users, between Dec 13 to Dec 30, 2009

Test Results

There's a lot more we can do with this data. If you have questions about it, ideas for how to analyze it, or derived metrics that you would be interested in seeing, please join the Test Pilot Discussion Group . If you generate any statistics or charts from the data, we would of course look forward to seeing those posted in the discussion group as well.

How to read the data

Data is in CSV format. The first line of the file is the header and gives the names of all the columns. After that, each line contains a summary of the data for a single user. Here are the column names and the explanation of each one:

Column Name Explanation
how_long_used_firefoxResponse to the survey question: How long have you used Firefox? (See below for response codes)
skill_levelResponse to the survey question: How would you describe your computer/web skill level? The answer is on a scale from 1 (least skilled) to 10 (most skilled). A value of -1 means the user declined to answer.
time_on_webResponse to the survey question: How much time do you spend on the Web per day? (See below for response codes)
ageResponse to the survey question: How old are you? (See below for response codes)
sexResponse to the survey question: Your gender? (See below for response codes)
fx_versionString describing the version of Firefox user was running, e.g. "3.5.5"
osString describing the operating system the user was running, e.g. "WINNT Windows NT 5.1"
localeString describing the Firefox language/locale setting, e.g. "en-US"
num_extensionsNumber of add-ons the user has installed
num_downloadsNumber of files that the user downloaded during the week of the study. (The same file downloaded multiple times will be counted each time.)
bookmarksNumber of bookmarks that the user had at the end of the week of the study. Note that livemark folders / RSS feeds do not contribute to this total. However, due to a bug, separators and smart folders do get added to this total, meaning that it is slightly higher (6-7 higher in most cases) than the true count.
foldersNumber of bookmark folders that the user's bookmarks are organized into, at the end of the week of the study
depthMaximum depth of bookmark folder hierarchy that the user's bookmarks are organized into, at the end of the week. E.g. if the value is 2, it means the user has at least one folder inside another folder.
num_sessionsNumber of separate Firefox sessions the user ran during the week. Increases by one each time the user starts up Firefox.
avg_session_lengthA number of milliseconds giving the average length of those Firefox sessions, e.g. milliseconds between startup and shutdown events. (Note that if no shutdown event was logged, the next startup event will be used as the ending time of the session; this may not be accurate especially in case where Firefox may have crashed.)
total_session_lengthTotal number of milliseconds of all sessions put together: equal to num_sessions multiplied by avg_session_length.
