Menu Item Usage: Analysis

Here are some analysis from the Menu Item Usage Study. The graphs on this page are based on the data submissions we received from about 8600 Test Pilots running Firefox versions 3.5 and above, between Feb.3rd and Feb.28th, 2010.

Contents:

  1. Which menu items are the most or the least commonly used?
  2. What are most and least used Firefox menu items by only mouse interactions?
  3. Which menu items are the most or the least commonly used, by mouse click or keyboard shortcut?

These analysis are generated based on early raw data, and they may not be super definitive or representative for all Firefox users. We are looking for help to firm them up! Please join the Test Pilot discussion group if you have any suggestions or questions.

Also, if you would like to do your own analysis of the data from this study, you can download samples of it here.

Question 1: Which menu items are the most or the least commonly used?

Note: This data is based on both mouse clicks and keyboard shortcuts. Menu items that are triggered by right-click on the mouse or by function key are not included.

By simply aggregating the total number of menu item clicks for all users, we get the above graph. It shows just that, presenting each menu item’s relative use for all UI methods (both mouse and keyboard shortcuts). We can see some justification for a condensed toolbar as many of the items are used very infrequently compared to the other menu items. For example, the menu items from “Page Setup” to “Character Encoding/UTF-16″ each make up less than 0.01% of the total menu bar clicks.

Question 2: What are most and least used Firefox menu items by only mouse interactions?

Note: Since menu bars are designed for mouse use, it is more relevant to look at item usage for just the mouse UI method (excluding keyboard shortcuts). This data is based on number of menu item clicks for all users. Menu items that are triggered by right-click on the mouse is not included.

This visualization presents information on two levels: the area of the circles are proportional to the total number of clicks for the menu group as a whole, and the slices correspond to the share of clicks for each item within the menu group. Bookmarks and Edit are by far the most utilized menus, representing over 70% of total clicks.

The high use of the bookmarks menu is somewhat surprising; an obvious problem of looking at aggregated data like this is the potential for outliers to skew the data. It will be interesting to delve into this issue more in depth and determine if the Bookmark menu (and other menus and menu items) is genuinely an important menu group for all users, or if the high usage is driven by a set of relatively few users who interact with the Bookmark menu extremely frequently.

Question 3: Which menu items are the most or the least commonly used, by mouse click or keyboard shortcut?

Examining the data in this way presents a slightly different picture: the top 5 most commonly used menu items are now “User Bookmark Item”, “Copy”, “Paste”, “Add-Ons”, and “Back”. In addition to “Add-Ons”, “Options” and “Bookmark This Page” are newly part of the top 10, replacing “Find”, “Open Location”, and “Find Again”.

Again these changes simply result from eliminating keyboard shortcut clicks and help us distinguish between mouse driven menu items and keyboard driven items. For example, by comparing the mouse UI chart (right) with the original all UI chart (left) we can clearly see that “New Tab” and “Close Tab” are predominately driven by keyboard shortcuts (as expected) and may not be the two most critical items to a mouse oriented toolbar (as suggested by the original chart).
